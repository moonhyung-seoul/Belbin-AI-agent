# 벨빈 AI Agent 개발 정리

## 1. 한 줄 요약 (Agent 미션)
"Belbin 진단 + Team Context Data를 통합 분석하여, FT의 사전 준비 시간을 단축하고(효율), 팀 맞춤형 인사이트/워크숍 설계/실행 넛지까지 연결해 '지속되는 변화'를 만드는 Team OD AI Agent."

## 2. 목표
- 효율성: 사전 내부 준비 단계 생산성 향상(진단 포함 자료 분석 및 팀 맞춤형 프로그램 설계)
- 효과성: 깊이 있는 분석 보고서 제공 + 임팩트 있는 액션 도출 지원 + 액션 실행을 돕는 넛지 제공

## 3. 사용자 및 사용 시나리오 (Who / When)
1) FT/OD 컨설턴트가 워크숍 사전 준비 기간에 Belbin 개인/팀 보고서와 Team Context Data를 업로드해 통합 분석 리포트와 맞춤 워크숍 아젠다 초안을 생성
2) FT/OD 컨설턴트가 워크숍 최종 점검에 참여자 구성/팀 발달단계/핵심 이슈를 반영해 Plan B 활동 대체안과 진행 스크립트 자동 추천
3) 팀 리더가 워크숍 오프닝 직전에 팀 강점/리스크 1장 요약 리더 브리프(중립 언어) 확인 후 오프닝 메시지에 반영
4) 팀 전 구성원이 워크숍 중반(보고서 해석 이후)에 개인 강점과 팀 맥락을 연결한 협업 기여 행동 팁을 받아 1~2개의 실천 행동 선택
5) FT가 워크숍 말미(액션 도출 단계)에 팀 병목/성과 목표와 연결된 Action Plan 템플릿(예: Top3 Team Moves + DoD/Owner/Partner/First step)으로 합의 구조화
6) 팀 리더가 워크숍 종료 직후(48시간 이내) 자동 생성된 팀 액션카드 요약 및 역할/책임 확인 요청 수신, 실행 공식화
7) 팀 구성원이 실행 정착 기간에 Teams/Slack 주간 체크인을 받고 진행/장애물/다음 행동 답변, Agent는 개인 맞춤 넛지와 팀 집계 현황 제공
8) 리더/HRBP가 워크숍 종료 후 매주 1장 브리프로 실행률/응답률/막힘 유형/지원 요청을 받아 즉시 지원 제공
9) HR/조직장이 2~3개월 후 임팩트 스냅샷(변화 근거 + 다음 개입 제안)으로 확산 여부 결정
10) 멀티 팀 진행 시 HR이 팀 강점과 다양한 정보를 조직 개편/후계자 계획에 참고
11) 동일 팀 재 디브리핑/워크숍 시 next step으로 더 효과적인 워크숍 설계 추천

### 3.1 출시 대상 / MVP 범위 / 개발 인력
- 출시 대상: FT + 워크숍 참여 구성원 전체(구성원 팀 리포트 접근 포함)
- MVP 단계: MVP-1 분석 리포트 생성(팀/개인/리더/FT 4종) → MVP-2 워크숍 설계 → MVP-3 실행 넛지
- 개발 인력: 2명

## 4. 입력 (Input)
### 4.1 입력 유형
- 파일 업로드: 개인/팀 Belbin 보고서(PDF), Team Context Data, 인터뷰 노트, 이미지/조직도(PNG/JPG), 기존 워크숍 아젠다/소개서(PPT)
- 폼 입력: 팀 기본 정보(팀 미션/업무/주요 이슈/제약/이해관계자)
- 선택 입력(향후): Jira/Confluence/Notion 링크, 회의록 링크

### 4.2 필수 입력
- 팀 Belbin Team Report PDF
- 개인 Belbin Individual Report PDF(전체 대상자)
- Team Context Data: 팀 업무/미션/인원 정보(이름, 직책/직급, 최근 조인/승진 여부, 개인 업무, 조직도, KPI/입력), 리더 고민사항(팀 상황/이슈) 등

### 4.3 팀 정보(확장/보완)
- 팀의 업무
- 인원 정보(이름, 직책/직급, 최근 조인/승진 여부, 주업무)
- 리더의 고민사항(팀 상황/이슈)
- 팀의 특수 상황(if any)
- 조직 진단 결과(if any)

## 5. 출력 (Output)
### 5.1 Team member pack (구성원 배포용)
1) 팀 보고서 연계 개인 분석자료
- Self vs Observer 비교 요약(일치/불일치 포인트, 해석)
- 개인 핵심 내용 요약(Top 역할, Bottom 역할, 특징 키워드)
- Team Role Balancer 관점의 개인 기여 포인트(Observed contribution)
  - 팀의 취약 역할을 개인 강점으로 보완 가능한 부분
  - 팀에서 허용 가능한 약점(몰린 강점의 과발현)을 개인 강점으로 완충/보완 가능한 부분
  - "어떤 상황에서 어떤 행동으로 팀 성과에 기여하는지"를 업무 장면으로 설명
- 주의/리스크 신호(과다사용/미사용, 스트레스 시 패턴 등)

2) 개인별 팀 기여 행동 액션 제안
- 강점을 더 강하게 발휘할 액션 2~3개
- 강점 역할의 허용 가능한 약점을 관리하기 위한 액션 2~3개
  - 8~9위 팀역할 보완에 대한 분석/행동 제안은 하지 않음
  - 도움 요청 대상/문장/방법 포함(요청 템플릿)
- 필드 구조 예시: 목적(팀 목표/업무 병목 연결) / 실행 행동 / 빈도 / DoD(완료 기준) / 파트너 추천 / 예상 효과

### 5.2 Leader pack (리더 배포용)
1) 팀 보고서 연계 리더 개인 분석자료
- Self vs Observer 비교 요약(팀 보고서 맥락)
- 리더의 현재 기여 포인트(팀 운영 영향 중심)
- 리더의 리스크/블라인드 스팟(결정 구조, 위임, 갈등 구조화, 정보 탐색 등)
- 팀의 부족 역할을 "사람이 아니라 시스템"으로 보완하는 리더 행동 가이드

2) 사후 리더 코칭 사전자료(FT/리더 공용 옵션)
- 사전 인터뷰 이슈(대화 부족, 역할 불명확, 갈등 회피 등)와 Belbin/팀 컨텍스트 연결 분석 메모
- 코칭 대화 구성: 이슈 정의 → 근거(관찰/데이터) → 가설 → 코칭 질문 → 실험 과제(2주) → 체크인 문항

### 5.3 Workshop pack (팀 워크숍 활용 자료)
1) 팀 인사이트 카드(1~2p 또는 슬라이드 1~2장)
- 팀 강점/약점/리스크(블라인드스팟 포함) 요약
- 팀 컨텍스트(미션, 고객, KPI, 업무흐름, 의사결정, 핸드오프)와 연결된 해석 문장
- "우리 팀이 잘할 때/무너질 때" 대표 장면(시나리오) 1~2개

2) 팀 공동 액션 안건 Long List(토론용)
- 팀 병목/리스크별 액션 후보
- 분류 예시: 의사결정 보완 / 갈등 보완 / 스트레스 보완 / 특정 역할 부재 보완
- 각 안건은 "왜(근거)? 무엇? 어떻게? 기대효과"까지 최소 정보 포함

### 5.4 대팀(Leadership team) 옵션
- 리더십팀(팀장 + 파트장들) 보고서/분석 별도 제공
- 리더십팀 관점 팀역할 분포/강점/약점/리스크 분석
- 팀장/파트장 관점에서 각 팀 구성원의 강점 활용 인사이트 제안
- 조직 상황별 대응 가이드(리더십팀 분석 vs 파트 단위 vs 혼합) 판단 기준
- 추가 입력(필요 시): 파트 간 의존성, 의사결정 라인, KPI 연계 구조, 갈등 이슈 등

### 5.5 FT-only pack
1) 프로그램 진행 시 주의사항(팀 맞춤)
- 팀 역할 편중에 따른 퍼실리테이션 리스크 및 대응
  - 예: ME/CF 강한 팀 → 발산(PL)과 평가(검증) 단계 분리 필요
  - 예: TW 강한 팀 → 필요한 갈등을 구조화(SH)하는 장치 필요
- 현장 대응 가이드: 침묵/방어/과열/논점 이탈/합의 실패 시 대체 질문/대체 활동

2) 참석자별 특이사항 1-pager
- 프로파일 유형
- Gap 큰 팀역할(Self vs Observer 차이 큰 역할)
- Top3 역할 & 점수 / Bottom1 역할 & 점수
- 부정 bar 30% 이상인 역할
- 부정 키워드 요약
- Drop point(과다 사용 시 영향, 스트레스 반응, 회의 패턴 등)

3) 워크숍 설계 패키지(FT용)
- 맞춤 아젠다(분 단위)
- 활동 설명(왜 필요한지)
- 질문 카드(ORID/Top-focused)
- 주의점/리스크 대응 스크립트

## 7. 실행 단계 UX/UI 플로우 (Step 0~8)
- STEP0 워크숍 종료 & 실행 전환: 워크숍 종료, 퍼실리테이터가 "바로 실험할 행동 1가지" 제안, ActionCard 생성 진입
- STEP1 개인 진입(웰컴): 개인 메시지/이메일 발송, 환영 메시지, ActionCard 만들기 CTA
- STEP2 ActionCard 생성: 사용자가 ActionCard 내용 입력
- STEP3 개인 실행 조건 설정: 실행 기간, 목표 수준, 리마인드 주기 설정, ActionCard 발송
- STEP4 실행 중 동행: 진행률 업데이트(0~100%), 주기 리마인드 발송, 30일 미업데이트 시 Gentle Reminder, "도와주세요/도와줄게요" 협업 입력
- STEP5 완료: 100% 완료 입력, 축하/격려 메시지 자동 발송
- STEP6 다음 실행 제안: AI가 다음 액션 제안, Accept/No Thanks
- STEP7 팀 실행 가시화: 팀 대시보드 요약 표시(개인 상세는 비공개)
- STEP8 리더 요약 & 리플렉션: 리더 1-page 실행 요약 전달, 4~6주 후 팀 리플렉션, ActionCard 종료 후 요약 페이지 전달

## 8. 학습/컨텍스트(지식) 소스
- Belbin 9가지 팀역할 정의/강점/용인 가능한 약점/용인 불가 약점(메르디스 벨빈 "팀이란 무엇인가")
- 벨빈 팀보고서 해석 포인트
- 개인보고서 해석 포인트(자가 vs 관찰자 일치도: 4가지 프로파일 유형)
- 샘플 보고서(팀 보고서 5개 + 개인 보고서)
- 관찰자 피드백 긍정/부정 키워드 및 설명 문구(파일)
- 팀역할 강화를 위한 Do & Don'ts(파일)
- 9가지 팀역할에서 받을 수 있는 도움(벨빈 UK SNS)

## 9. 분석/해석 포인트 (Belbin)
### 9.1 팀 보고서 해석 포인트
- Team Role Balance: 팀역할 부재/과잉, 팀원들의 최약점(개인 9위 팀역할 분포)
- 팀역할 평균 점수 순위(상위 1~3위, 하위 8~9위)
- 개인 최고점과 팀 평균 간 Gap
- 관찰자 피드백 종합
- 팀 업무와 연계된 해석 필수
- 팀 리더 역할은 별도 해석 필요

### 9.2 개인 보고서 해석 포인트
- 자가 응답 vs 관찰자 응답 일치도: 4가지 프로파일 유형

## 10. 팀+개인 결합 인사이트 아이디어
- 팀 역할 구성 개요(자가 vs 관찰자 일치도): 진짜 강점 인식 정도
- 상위 2개 역할/팀 평균과의 Gap를 팀 미션과 연결해 심각도 분석
- 개인-팀 비교 기반 핵심 인물 분석(단일 실패점, 미인식 강점 보유, 숨은 화합자 등)
- 관찰자 응답 개괄을 팀 미션과 연결해 팀 문화 강점/약점 분석
- Overall 팀 미션 수행 적합성(100점 기준), 위협 요소/영향 분석, 전략적 개입 로드맵, 측정/모니터링 체계, 성공/실패 시나리오
- 리더-구성원 시너지/갈등 분석: 개인 보고서(1~4위/9위/관찰자 키워드) + 팀 미션 기반
  - 시너지/갈등/관계 관리 전략 도출
  - 갈등 발생 시나리오, 개선 방향, 코칭 방향성, 팀원별 맞춤 제안, 팀 차원 구조 개선 제안
  - Belbin 업무관계 보고서는 유용성 낮음 가설: 리더/미션 입력 + 8~9위 고려 필요

### 10.1 레포트 인사이트 강화 요소(추가 시 인사이트 상승)
1) "팀 문화 한 문장" + 핵심 경향 요약
- 예: "우리 팀은 CF/CO 중심이라 품질·안정·조율 강점이 크다. 대신 속도/실험/도전 의사결정이 느려질 수 있다."
2) "팀의 블라인드 스팟(과다 사용 리스크)" 페이지
- 역할 과다 → 행동 패턴 → 팀 영향 → 발생 장면
3) "팀이 놓치기 쉬운 관점(부재 역할의 비용)" 섹션
- 약한 역할 1~2개 + 해당 역할 부재 시 실제 문제 + 보완 전략
4) "갈등/오해 조합(관계 리스크 맵)"
- SH↔TW, PL↔CF, ME↔IMP 등 충돌 포인트
- 오해 이유(의도) / 표현 방식(행동) / 상호 번역 문장(한 줄 코칭)
5) "의사결정 스타일/리스크" 페이지(Vroom–Yetton/DiSC 스타일)

## 11. Team Context Data (추가로 필요한 정보)
1) 성과 정의 & KPI 구조
- Top KPI 2~3개, 리드/래그 지표, 최근 6개월 성과 흔들린 지점
2) 핵심 업무 흐름(Workflow) & 핸드오프(Interface)
- 핵심 프로세스 3~5개(입력→처리→출력), 병목/재작업 구간, 인터페이스 관계
- 질문 예시: "업무가 멈추는 지점은 어디인가?" "재작업 이유는?"
3) 의사결정 구조(Decision Rights) & 기준(Standard)
- 느린 결정, 합의/승인/리스크 요인, RACI/DACI, DoD 존재 여부
- 질문 예시: "가장 오래 걸리는 결정 1가지는?" "완료 기준은?"
4) 고객(내/외부) & Value Proposition
- 주요 고객 2~3 그룹, 고객 기대, VoC, 팀 차별점
- 질문 예시: "팀이 없으면 고객이 가장 불편한 점은?" "고객이 요구하는 품질/속도/정확성은?"
5) 현재 이슈 유형화(관계 vs 과업 vs 구조)
- 관계 이슈: 신뢰/피드백/갈등 회피
- 과업 이슈: 우선순위/역할 혼선/품질 기준
- 구조 이슈: 리소스/권한/타부서 의존
- 질문 예시: "반복되는 문제는 사람/관계인가, 일/구조인가?"

## 13. 실행 넛지/코칭 AI 설계 메모
- 기본 프레임: 목표 설정 → 진행 추적 → 스마트 넛지
- 목표를 추적 가능한 형태로 구조화
  - Milestones: 2주/4주/8주 체크포인트
  - Habits: 주 1~2회 작은 반복 행동
- 진행 추적은 점수표보다 짧은 체크인 루프
  - 주간 체크인(1~3분): 실행 여부/장애물/다음 주 작은 행동
  - 자기평가(Self-assessment): 전후 비교로 성장 가시화
  - 관리자/HR 대시보드: 개인 내용 비공개, 집계 패턴만
- 넛지 유형
  1) 시간 기반(정기 리듬)
  2) 이벤트 기반(회의 직전/직후, 캘린더 연동)
  3) 행동 기반(미실행/중단 감지)
  4) 코칭 기반(역할극/연습)
- 워크숍 액션 플랜 자동 푸쉬 권장 조합
  - 기준 + 다음 행동 구조화
  - 주간 1~3분 체크인
  - 캘린더/Teams/Slack 기반 상황 넛지
  - 관리자에게는 집계 대시보드만
