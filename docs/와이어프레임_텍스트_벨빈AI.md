# 벨빈 AI Agent - 화면 흐름/와이어프레임 텍스트 (MVP-1: 분석 리포트)

## 0. 전체 플로우 요약
- FT: 프로젝트 생성 → 데이터 업로드 → 분석 초안 생성 → 편집/승인 → PDF 확정 → 공유 링크 발송
- 리더: 팀 리포트 + 리더용 리포트 확인
- 구성원: 팀 리포트 + 개인 리포트 확인

---

## 1. 로그인/권한 진입
### 1.1 로그인
- 입력: 이메일/비밀번호
- 보조: 비밀번호 재설정
- 결과: 역할별 대시보드로 이동(FT/리더/구성원)

---

## 2. FT 대시보드
### 2.1 프로젝트 리스트
- 카드: 팀명, 상태(업로드/분석/승인/배포), 최근 수정일
- 액션: [새 프로젝트 생성], [프로젝트 열기]

### 2.2 프로젝트 상태 요약
- 업로드 완료율, 리포트 생성 상태, 승인 상태, 배포 상태

---

## 3. 프로젝트 생성
### 3.1 기본 정보 입력
- 필드: 프로젝트명, 팀명, FT 담당자
- 액션: [다음]

### 3.2 팀 컨텍스트 입력
- 필드: 팀 미션/업무, 주요 이슈, 제약/리스크, 이해관계자
- 필드(선택): KPI, 고객(내/외부), 의사결정 구조
- 액션: [저장]

---

## 4. 데이터 업로드
### 4.1 업로드 섹션
- 팀 Belbin Team Report PDF
- 개인 Belbin Individual Report PDF(전체 대상자)
- Team Context Data(XLSX)
- 선택: 인터뷰 노트, 조직도 이미지, 기존 워크숍 자료(PPT)

### 4.2 업로드 상태
- 파일명, 업로드 시간, 파싱 상태(대기/완료/실패)
- 액션: [재업로드], [파싱 로그 보기]

---

## 5. 분석 초안 생성
### 5.1 생성 버튼
- CTA: [통합 분석 초안 생성]
- 진행 표시: 처리 단계(추출 → 매핑 → 초안 생성)

### 5.2 결과 미리보기
- 섹션: 팀 리포트 / 팀원용 개인 리포트 / 리더용 리포트 / FT-only 리포트
- 각 섹션에 근거 표시(입력 자료/페이지)
- "확인 필요" 라벨 표시

---

## 6. 리포트 편집/승인 (FT)
### 6.1 편집 화면
- 블록 기반 편집(섹션 단위)
- 수정 가능한 요소: 텍스트, 표, 강조 문장
- 수정 이력 및 원문 비교(토글)

### 6.2 승인/확정
- 액션: [승인 후 PDF 생성]
- 결과: 최종 PDF 다운로드 + 공유 링크 생성

---

## 7. 리포트 패키지 배포
### 7.1 공유 링크 발송
- 대상: 리더/구성원
- 권한: 리더용/팀원용/팀 리포트 구분
- 채널: 이메일 + 인앱 알림

---

## 8. 리더 리포트 뷰
- 탭: 팀 리포트 / 리더용 리포트
- 내용: 팀 강점/리스크 요약 + 리더 분석/리스크/행동 가이드
- 액션: [PDF 다운로드]

---

## 9. 구성원 리포트 뷰
- 탭: 팀 리포트 / 개인 리포트
- 내용: 개인 Self vs Observer, Top/Bottom 역할, 기여 포인트, 행동 제안
- 액션: [PDF 다운로드]

---

## 10. 알림센터
- 이벤트: 리포트 생성 완료, 승인 완료, 공유 링크 발송
- 채널: 인앱 + 이메일

---

## 11. 관리/설정(FT)
- 권한 관리(FT/리더/구성원)
- 데이터 보관/삭제 정책(관리자 권한)
- 프로젝트 아카이브

---

## vNext 참고(워크숍/실행)
- 워크숍 설계(아젠다/활동/질문 카드/스크립트)
- ActionCard 생성/체크인/리마인드
