# 리포트 템플릿 구조 (MVP-1)

## 0. 공통 원칙
- 톤: 중립적/비판 없는 언어(“문제” 대신 “리스크/패턴/개선 기회”)
- 근거 표시: 모든 핵심 주장/수치에 입력 근거 표시
- 근거 표기 형식(예): [SRC: TeamReport p.3], [SRC: IndividualReport 홍길동 p.5], [SRC: TeamContext KPI]
- 데이터 부족 시: "확인 필요" 태그 부착
- 개인 정보 보호: 개인 상세는 FT 전용, 리더/구성원은 권한 범위 내 열람

## 1. 접근 권한 요약
- 팀 리포트: 리더/구성원/FT
- 개인 리포트(팀원용): 해당 구성원 + FT
- 리더 리포트: 리더 + FT
- FT-only 리포트: FT 전용

## 2. 공통 메타 섹션(모든 리포트 상단)
- 프로젝트명 / 팀명 / 생성일 / 버전
- 입력 자료 목록(팀 보고서/개인 보고서/Team Context Data/인터뷰 노트)
- 작성자(Agent) / 검수자(FT) / 승인일

---

# A. 팀 리포트(Team pack)
## A-1. 목적
- 팀 전체의 강점/약점/리스크 및 액션 후보를 요약

## A-2. 섹션 구조
1) 팀 요약
- 팀 미션/업무 요약 (Team Context 기반)
- 팀의 핵심 성향 한 문장 요약(예: CF/CO 중심)

2) 팀 강점/약점/리스크
- 강점 Top 2~3
- 약점/리스크 Top 2~3(블라인드스팟 포함)
- 팀 역할 밸런스 요약(부재/과잉)

3) 팀 컨텍스트 연결 해석
- 미션/KPI/업무흐름/의사결정과 연결된 해석 문장

4) "잘할 때 / 무너질 때" 시나리오
- 대표 장면 1~2개

5) 팀 공동 액션 안건 Long List
- 범주: 의사결정/갈등/스트레스/역할 부재 보완
- 각 안건 필드: 왜(근거) / 무엇 / 어떻게 / 기대효과

## A-3. 필드 정의(요약)
- 팀 성향 한 문장: 텍스트 1~2문장
- 강점/리스크: 불릿, 근거 필수
- 액션 안건: 템플릿 기반 문장 + 근거

---

# B. 개인 리포트(팀원용)
## B-1. 목적
- 개인의 강점과 팀 맥락 기반 기여 포인트, 실천 행동 제안 제공

## B-2. 섹션 구조
1) Self vs Observer 요약
- 일치/불일치 포인트
- 프로파일 유형(4가지 중 분류)

2) 핵심 내용 요약
- Top 역할 / Bottom 역할
- 특징 키워드

3) 기여 포인트(Observed contribution)
- 팀의 취약 역할 보완 포인트
- 몰린 강점의 과발현을 완충하는 포인트
- "어떤 상황에서 어떤 행동으로 기여하는지" 업무 장면 서술

4) 주의/리스크 신호
- 과다 사용/미사용, 스트레스 반응 패턴

5) 팀 기여 행동 제안
- 강점 강화 액션 2~3개
- 허용 가능한 약점 관리 액션 2~3개
- 도움 요청 포함(대상/요청 문장)

## B-3. 필드 구조(액션 제안)
- 목적(팀 목표/업무 병목 연결)
- 실행 행동
- 빈도
- DoD(완료 기준)
- 파트너 추천
- 예상 효과

## B-4. 주의 규칙
- 8~9위 팀역할 "보완"을 직접 제안하지 않음
- 개인의 강점에서 파생되는 허용 가능한 약점을 관리하는 방식으로 제안

---

# C. 개인 리포트(리더용)
## C-1. 목적
- 리더의 팀 운영 기여/리스크를 팀 보고서 맥락에서 요약

## C-2. 섹션 구조
1) Self vs Observer 요약(리더)
2) 현재 기여 포인트(팀 운영 영향 중심)
3) 리스크/블라인드스팟
4) 시스템으로 보완하는 리더 행동 가이드
5) (옵션) 사후 리더 코칭 사전자료
- 이슈 정의 → 근거 → 가설 → 코칭 질문 → 2주 실험 → 체크인

---

# D. FT-only 리포트
## D-1. 목적
- 현장 퍼실리테이션 리스크 관리 + 참석자 특이사항 요약

## D-2. 섹션 구조
1) 프로그램 진행 시 주의사항(팀 맞춤)
- 예: ME/CF 강한 팀 → 발산/평가 단계 분리
- 예: TW 강한 팀 → 필요한 갈등 구조화 장치

2) 참석자별 특이사항 1-pager(개인 카드)
- 프로파일 유형
- Gap 큰 팀역할(Self vs Observer 차이 큰 역할)
- Top3 역할 & 점수 / Bottom1 역할 & 점수
- 부정 bar 30% 이상인 역할
- 부정 키워드 요약
- Drop point(과다 사용 영향/스트레스 반응/회의 패턴)

---

# E. 출력/편집/승인 규칙
- 출력 순서: 초안 생성 → FT 편집 → FT 승인 → PDF 확정
- 편집 가능 범위: 텍스트/표/강조 문장(근거 태그 유지)
- 변경 이력 저장(FT 수정 내용 로그)

---

# F. 필수 입력 매핑(요약)
- 팀 리포트: Belbin Team Report + Team Context Data
- 개인 리포트(팀원/리더): Individual Report + Team Report 요약 + Team Context
- FT-only: Team/Individual Report + 관찰자 키워드/부정 bar 파일

---

# G. 파일명 규칙(권장)
- TeamReport_{팀명}_v{버전}.pdf
- MemberReport_{이름}_v{버전}.pdf
- LeaderReport_{리더명}_v{버전}.pdf
- FTOnly_{팀명}_v{버전}.pdf
