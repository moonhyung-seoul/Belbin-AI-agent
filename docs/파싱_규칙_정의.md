# 파싱 규칙 정의 (MVP-1)

목적: Belbin 팀/개인 PDF와 Team Context에서 **리포트 스키마 필드**를 안정적으로 추출·계산하기 위한 규칙을 정의한다.

---

## 0) 입력/전제
- 입력 파일
  - 팀 보고서 PDF (Team Report)
  - 개인 보고서 PDF(전체 대상자)
  - Team Context (엑셀/폼)
  - 관찰자 키워드 분류표(부정/긍정) — **이 표 기준으로 부정 키워드 판단**
- 팀역할 코드: `RI, TW, CO, PL, ME, SP, SH, IMP, CF`
- 팀역할 코드-한글명 표준 매핑
  - PL: 창조자
  - RI: 자원탐색가
  - CO: 지휘조절자
  - SH: 추진자
  - ME: 냉철판단자
  - TW: 분위기조성자
  - IMP: 실행자
  - CF: 완결자
  - SP: 전문가
- 개인 최종점수: **자가 1/3 + 관찰 2/3**

---

## 1) 텍스트 추출 규칙
- 기본: `pdftotext -enc UTF-8 -layout` 사용
- 페이지 경계: 텍스트 내 `\f`로 분리
- 소스 표기용 페이지: 추출 시 페이지 인덱스(1-based) 기록

---

## 2) 팀 보고서 파싱

### 2-1. 섹션 앵커
- 팀역할 구성 개요
- 개인-팀 비교
- 팀역할 평균
- 관찰자 응답 개괄
- 팀역할의 좋은 예
- 팀역할 제안

### 2-2. 팀 평균(Team role average)
- 섹션: **팀역할 평균**
- 결과: 역할별 팀 평균 점수(0~100)

### 2-3. 관찰자 응답 개괄
- 섹션: **관찰자 응답 개괄**
- 키워드/빈도 추출
- **부정 키워드 Top3 선정**: 관찰자 키워드 분류표의 *부정* 항목만 대상으로 빈도 높은 3개
  - 동률 발생 시: 빈도 동일한 후보 모두 보관 → FT가 최종 3개 선택

### 2-4. 개인-팀 비교(팀 리포트 표시용)
- 개인 최종점수(개인 보고서에서 계산) + 팀 평균을 결합
- 표시 기준:
  - **표시**: `개인 최종점수 >= 팀평균 * 1.35`
  - **강조**: `개인 최종점수 >= 팀평균 * 2.0`
  - **팀 평균=0**: 비교 계산 생략, `부재 역할` 표기. 개인 점수>0이면 `희소 역할 보유` 플래그
- 팀 평균 2종 계산:
  - 방법 A: 개인 최종점수 평균
  - 방법 B: 팀 보고서 차트 OCR(선택)
  - 두 값 차이 발생 시: **FT 선택 전 출력 보류**

---

## 3) 개인 보고서 파싱

### 3-1. 개인 리포트 구분
- 앵커: `Belbin 개별 대상자 팀역할보고서`
- 각 리포트별 `이름`, `관찰자 수`, `자가진단 완료일` 추출

### 3-2. 자가/관찰 점수 추출
- 섹션: **자가진단과 관찰자진단 비교**
- 포맷 예시:
  - `ME 92 70` (자가 92, 관찰 70)
- 역할별 `selfScore`, `observerScore` 확보

### 3-3. 개인 최종점수 계산
- `finalScore = (selfScore * 1/3) + (observerScore * 2/3)`
- **Top1~3 역할**: finalScore 기준 내림차순
- **Bottom 역할**: finalScore 최하위 1개

---

## 4) 리포트 생성 규칙(필수)

### 4-1. 팀 리포트
- 팀 성향 한 문장
- 팀 강점/리스크 Top 2~3
- 역할 밸런스(부재/과잉)
- 개인-팀 비교(위 기준)
- **팀 강점 발휘 순간/팀 잠재 이슈 순간** 시나리오 4~5개
- 팀 공동 액션 안건 Long List: **3개**
  - “왜/무엇/어떻게(옵션 2~3)/필요한 행동·팀역할”
  - 관찰자 응답 개괄의 **부정 키워드 Top3** 고려
  - 기대효과/부정키워드 항목은 출력에 포함하지 않음

### 4-2. 개인/리더/FT-only 리포트
- 기존 스키마 기준 준수
- 근거가 부족한 항목은 `needConfirm` 표시

---

## 5) 오류/예외 처리
- 섹션 누락 시: 해당 필드 `needConfirm = true`
- 점수 누락 시: 해당 역할 점수 `null`, 계산 제외
- 팀 평균=0 처리: 비교 계산하지 않고 부재/희소 플래그만 유지

---

## 6) 추적 가능성
- 모든 주요 필드에 `[SRC: 문서명 p.N]` 또는 `source` 객체 기록
