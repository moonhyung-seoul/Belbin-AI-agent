# 팀 리포트 자동 매핑 정의 (MVP-1)

목적: 파싱 결과(CSV/JSON)를 **팀 리포트 템플릿 필드에 자동 채움**하는 규칙을 명확히 한다.

---

## 1) 입력(파싱 결과)
- `team_average.csv` : 역할별 팀 평균(finalScore)
- `person_top3.csv` : 개인별 Top1~3 역할
- `highlighted_roles.csv` : 35% 이상 표시 / 200% 이상 강조 대상
- `team_report_sections` : 팀 보고서에서 추출된 텍스트 요약(강점/리스크/밸런스/시나리오 후보)
- `observer_negative_top3` : 키워드 분류표(부정) 기반 Top3

> 주의: 팀 평균 2종(개인 평균 vs OCR 평균) 차이 발생 시 **FT 선택 전 출력 보류**

---

## 2) 출력(팀 리포트 템플릿 필드)
팀 리포트 JSON 스키마 기준:
- `teamReport.teamPersonaOneLine`
- `teamReport.strengths`
- `teamReport.risks`
- `teamReport.roleBalance`
- `teamReport.personTeamComparison`
- `teamReport.contextInterpretation`
- `teamReport.scenarios`
- `teamReport.actionLongList`

---

## 3) 매핑 규칙(필드별)

### A. 팀 성향 한 문장
| 입력 | 변환 | 출력 |
|---|---|---|
| `team_report_sections.teamPersona` | 그대로 사용 | `teamReport.teamPersonaOneLine` |

### B. 팀 강점/리스크 Top 2~3
| 입력 | 변환 | 출력 |
|---|---|---|
| `team_report_sections.strengthCandidates` | 상위 2~3개 선택 | `teamReport.strengths[]` |
| `team_report_sections.riskCandidates` | 상위 2~3개 선택 | `teamReport.risks[]` |

### C. 역할 밸런스
| 입력 | 변환 | 출력 |
|---|---|---|
| `team_report_sections.roleBalanceSummary` | 그대로 | `teamReport.roleBalance.summary` |
| `team_report_sections.missingRoles` | 그대로 | `teamReport.roleBalance.missingRoles` |
| `team_report_sections.overrepresentedRoles` | 그대로 | `teamReport.roleBalance.overrepresentedRoles` |

### D. 개인-팀 비교(Top1~3 vs 팀 평균)
| 입력 | 변환 | 출력 |
|---|---|---|
| `person_top3.csv` | 개인별 Top1~3 역할 집계 | `teamReport.personTeamComparison.comparisons[].topRoles` |
| `highlighted_roles.csv` | 35% 이상만 표시 | `teamReport.personTeamComparison.comparisons[].items[]` |
| `team_average.csv` | 역할별 팀 평균 | `teamReport.personTeamComparison.teamAverages.calcFromIndividuals` |

**표시 규칙**
- 표시: `개인 최종점수 >= 팀평균 * 1.35`
- 강조: `개인 최종점수 >= 팀평균 * 2.0`
- 팀 평균 0: 비교 계산 생략, `teamAvgZero=true`, 개인 점수>0이면 `scarceRoleFlag=true`

### E. 팀 컨텍스트 연결 해석
| 입력 | 변환 | 출력 |
|---|---|---|
| `team_context` | 핵심 문장 2~3개 요약 | `teamReport.contextInterpretation[]` |

### F. 시나리오(팀 강점 발휘 순간/팀 잠재 이슈 순간)
| 입력 | 변환 | 출력 |
|---|---|---|
| `team_report_sections.scenarioCandidates` | 4~5개 선정 | `teamReport.scenarios[]` |

### G. 팀 공동 액션 안건 Long List (3개)
| 입력 | 변환 | 출력 |
|---|---|---|
| `team_report_sections.bottleneckCandidates` | 이슈 3개 선정 | `teamReport.actionLongList[]` |
| `observer_negative_top3` | 안건 문구에 반영 | `teamReport.actionLongList[].what/how/why` |

**형식 규칙**
- 각 안건은 `왜/무엇/어떻게(옵션 2~3)/필요한 행동·팀역할`을 포함
- 기대효과/부정키워드 항목은 **출력에 포함하지 않음**

---

## 4) 자동 매핑 결과(샘플 구조)
```json
{
  "teamReport": {
    "teamPersonaOneLine": {"text": "string"},
    "strengths": [{"title": "string", "detail": "string"}],
    "risks": [{"title": "string", "detail": "string"}],
    "roleBalance": {"missingRoles": [], "overrepresentedRoles": [], "summary": "string"},
    "personTeamComparison": {
      "teamAverages": {"calcFromIndividuals": []},
      "comparisons": [
        {"person": {"name": "홍길동"}, "topRoles": ["ME","CF","IMP"], "items": [{"role": "ME"}]}
      ]
    },
    "scenarios": [{"type": "strength_moment", "title": "string", "scene": "string"}],
    "actionLongList": [{"category": "decision", "why": "string", "what": "string", "how": "string", "neededRoles": ["CO","SH"]}]
  }
}
```

---

## 5) FT 개입 지점
- 팀 평균 2종 차이 발생 시 **FT 선택 전 출력 보류**
- 부정 키워드 Top3 동률 발생 시 **FT가 최종 3개 확정**
