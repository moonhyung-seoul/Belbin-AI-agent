# 리포트 데이터 양식(쉬운 버전)

## 목적
- 리포트에 들어갈 항목을 표준화해, 화면/출력/PDF가 흔들리지 않게 만드는 기준

---

# 1) 공통 정보(모든 리포트 상단)
- 프로젝트명
- 팀명
- 생성일 (FT 전용 표기)
- 버전 (FT 전용 표기)
- 입력 자료 목록(팀 보고서/개인 보고서/Team Context 등)
- 검수자(FT) / 승인일 (FT 전용 표기)

---

# 2) 팀 리포트(Team pack)
## 필수 항목
- 팀 미션/업무 요약 (FT 전용 표기)
- 팀 성향 한 문장
- 팀 강점 Top 2~3
- 팀 약점/리스크 Top 2~3 (팀 상황에 따라)
- 팀 역할 밸런스(부재/과잉 요약)
- 개인-팀 비교(Top1~3 vs 팀 평균, 개인 최종점수 기준)
  - 팀 평균 대비 **35% 이상 또는 200% 이상**인 인원/팀역할만 표시
  - 팀 평균 0일 때: 부재 역할 표기, 개인 점수>0이면 희소 역할 보유 플래그
  - 팀 평균 2종 계산 -> 차이 감지 시 FT 선택 후 출력
- 팀 컨텍스트 연결 해석
- "팀 강점 발휘 순간/팀 잠재 이슈 순간" 시나리오 4~5개
- 팀 공동 액션 안건 Long List (3가지 안건 권장)

## 액션 안건 형식
- 왜(근거)
- 무엇
- 어떻게
- 기대효과
- (권장) 이 팀의 병목으로 느껴지는 순간 + 그때 필요한 기대 행동 및 팀 역할
- (권장) 관찰자 응답 개괄의 부정 키워드 Top3를 고려

## 액션 안건 샘플 (3개)
1) 갈등 이슈 구조화
   - 왜(근거): 갈등을 두려워함 경향으로 핵심 이슈가 표면화되지 않음
   - 무엇: 갈등 이슈를 “사실→영향→요청” 구조로 정리
   - 어떻게(옵션)
     1. 주간 회의에 10분 이슈 정리 슬롯 고정
     2. 이슈는 “사실/영향/요청” 3줄 템플릿으로 제출
     3. 갈등 이슈는 반드시 결정/보류로 마무리
   - 필요한 행동/팀역할: TW, CO

2) 결정 기준 단순화
   - 왜(근거): 까다로움 경향으로 결정이 지연될 수 있음
   - 무엇: 의사결정 기준을 2~3개로 단순화
   - 어떻게(옵션)
     1. “효과/난이도/리스크” 3요소로만 판단
     2. 결정권자 1명을 지정하고 회의 전 공유
     3. 논의 시간 상한(타임박스) 적용
   - 필요한 행동/팀역할: CO, SH

3) 검증 루프 도입
   - 왜(근거): 회의적임(의심이 많음) 경향으로 합의가 지연될 수 있음
   - 무엇: “가설→근거→검증” 프레임으로 논의 구조화
   - 어떻게(옵션)
     1. 회의마다 가설/근거/검증 방법 1줄 기록
     2. 2주 실험 과제로 먼저 검증 후 확대
     3. 반대 의견은 “대안+증거” 형식으로 제시
   - 필요한 행동/팀역할: ME, PL

# 3) 개인 리포트(팀원용)
## 필수 항목
- Self vs Observer 요약(일치/불일치)
- 프로파일 유형(4가지 중 하나)
- Top 역할 / Bottom 역할
- 특징 키워드
- 기여 포인트(업무 장면 기반, 팀 밸런스와 연계된 인사이트 포함)
- 주의/리스크 신호(개인의 Top 역할에 대한 관찰자 피드백 기반의 허용 가능한 약점 + 팀 밸런스 연계 인사이트)

## 행동 제안(2~3개씩)
- 강점 강화 액션
- 허용 가능한 약점 관리 액션(도움 요청 포함)

## 행동 제안 형식
- 목적(팀 목표/병목 연결)
- 실행 행동
- 빈도
- DoD(완료 기준)
- 파트너 추천
- 예상 효과

---

# 4) 리더 리포트(리더용)
## 필수 항목
- Self vs Observer 요약
- 현재 기여 포인트(팀 운영 영향 중심)
- 리스크/블라인드스팟(Top 역할 관찰자 피드백 기반의 허용 가능한 약점 + 팀 밸런스 연계 인사이트)
- 시스템으로 보완하는 리더 행동 가이드
- (옵션) 사후 리더 코칭 사전자료

## 코칭 사전자료 형식
- 이슈 정의 -> 근거 -> 가설 -> 코칭 질문 -> 2주 실험 -> 체크인

---

# 5) FT-only 리포트
## 필수 항목
- 퍼실리테이션 주의사항(팀 맞춤)
- 참석자별 특이사항 1-pager
  - 프로파일 유형
  - Gap 큰 팀역할
  - Top3/Bottom1 역할 & 점수
  - 부정 bar 30% 이상 역할
  - 부정 키워드 요약
  - Drop point

# 6) 근거 표기 규칙(모든 리포트 공통)
- 모든 핵심 주장/수치에 근거 표시

- 근거 부족 시 "확인 필요" 표기
