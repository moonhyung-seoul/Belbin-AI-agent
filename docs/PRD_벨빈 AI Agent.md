# PRD - 벨빈 AI Agent (MVP-1: 분석 리포트)

## 문서 정보
- 문서명: PRD - 벨빈 AI Agent (MVP-1: 분석 리포트)
- 버전: v0.2
- 상태: Draft
- 작성일: 2026-01-29
- 대상: FT/OD 컨설턴트 + 리더/구성원(리포트 수신자)

## 1. 배경 / 문제 정의
- Belbin 진단과 팀 컨텍스트 데이터를 통합 분석하는 데 FT의 사전 준비 시간이 과다 소요됨
- 깊이 있는 해석과 팀 맞춤형 보고서를 일관되게 제공하기 어려움
- 워크숍 이후 실행 정착은 중요하지만, MVP-1에서는 리포트 품질/속도 확보에 집중

## 2. 목표
### 2.1 효율성
- 사전 내부 준비 단계(진단 포함, 자료 분석 및 팀 맞춤형 보고서 제작) 생산성 향상

### 2.2 효과성
- 깊이 있는 분석 보고서 제공으로 임팩트 있는 액션 도출 지원

## 3. 성공 기준
### 3.1 MVP-1(분석 리포트)
- 사전 준비 시간: 기존 대비 70% 감축
- 리포트 활용도: 수정 없이 60% 이상 사용

### 3.2 vNext(워크숍/실행)
- 워크숍 만족도: 90%
- 개인/팀 액션 실행률: 50%

## 4. 사용자 / 권한
### 4.1 역할 정의
- FT/OD 컨설턴트: 프로젝트 생성, 데이터 업로드, 리포트 생성 및 편집/승인, 배포 관리
- 팀 리더: 리더용 리포트 + 팀 리포트 확인
- 팀 구성원: 개인 리포트 + 팀 리포트 확인

### 4.2 기본 권한 원칙
- 개인 상세 정보는 FT 전용, 리더/HR에게는 익명 집계가 기본
- 리더는 팀 요약 및 리더용 리포트 열람 가능
- 구성원은 본인 리포트와 팀 리포트 열람 가능

## 5. 범위 (MVP-1)
### 5.1 In-Scope
- 분석 리포트 생성(팀/개인/리더/FT 전용 4종 패키지)
- Belbin 원문 보고서에 대한 추가 해석/요약 포함
- FT 편집/승인 후 PDF 확정
- 리포트 공유 링크/다운로드(권한 기반)

### 5.2 Out-of-Scope
- Belbin 진단 자체 생성/평가
- 보고서에 없는 사실 추측/단정
- 팀역할을 고정적 개인 특성으로 해석
- 고객사 보안 정책 위반(외부 전송/무단 저장)
- 워크숍 설계/프로그램 패키지 생성
- 실행 넛지(ActionCard/체크인/리마인드)

## 6. 입력 (Input)
### 6.1 필수 입력
- 팀 Belbin Team Report PDF
- 개인 Belbin Individual Report PDF(전체 대상자)
- Team Context Data(업무/미션/인원/조직도/KPI/리더 고민 등)

### 6.2 선택 입력
- 인터뷰 노트
- 조직도 이미지(PNG/JPG)
- 기존 워크숍 자료(PPT)
- 외부 링크(Jira/Confluence/Notion/회의록 등) - 향후

## 7. 출력 (MVP-1 Deliverables)
### 7.1 분석 리포트 패키지(4종)
#### 7.1.1 개인배포용(팀원용)
- 팀 보고서 연계 개인 분석자료(자가 vs 관찰자, 핵심내용 요약, 현재 기여)
- 팀 기여 행동 제안(강점 강화/허용 가능한 약점 보완, 도움 요청 포함)

#### 7.1.2 개인배포용(리더용)
- 팀 보고서 연계 리더 개인 분석자료(자가 vs 관찰자, 핵심내용 요약, 현재 기여)
- 리더 별도: 사후 리더코칭용 사전 인터뷰 이슈 연결 분석자료

#### 7.1.3 팀(Team pack)
- 팀 강점/약점/리스크 요약
- 팀 공동 액션 안건 제안 Long List

#### 7.1.4 FT-only pack
- 퍼실리테이터 주의 사항(팀 맞춤)
- 참석자별 특이사항 1-pager(프로파일 유형, Gap 큰 팀역할, Top3/Bottom1 역할 점수, 부정 bar 30% 이상 역할, 부정 키워드, Drop point)

## 8. 핵심 사용자 시나리오 (MVP-1)
1) FT가 프로젝트 생성 후 팀 데이터 업로드
2) Agent가 통합 분석 리포트 초안 생성
3) FT가 초안 편집/승인 후 PDF 확정
4) 리더/구성원에게 권한 기반 공유 링크 전달

## 9. 기능 요구사항 (MVP-1)
### 9.1 프로젝트/데이터 관리
- 프로젝트 생성/관리
- 파일 업로드(PDF/PPT/PNG/JPG/XLSX)
- 입력 폼(팀 기본 정보, 이슈, 제약 등)
- 업로드 자료 버전 관리 및 변경 이력

### 9.2 분석 리포트 생성
- Belbin 팀/개인 보고서 추출 및 정규화
- Team Context Data와 연계한 해석 문장 생성
- 리포트 구성 요소 템플릿화
- 근거 표시(어떤 입력 자료의 어떤 항목에 기반했는지)
- FT 편집/승인 플로우: 편집 가능한 초안 → FT 수정 → 승인 후 PDF 확정

### 9.3 리포트/패키지 출력
- PDF 다운로드
- 공유 링크(권한 기반)

## 10. 비기능 요구사항
### 10.1 보안/개인정보
- 프로젝트 단위 데이터 격리
- 접근 권한/감사 로그
- 데이터 보관/삭제 정책: 보관 기간 12개월 후 아카이브(접근 차단), 관리자 복구 가능(복구 가능 기간: 무기한), 삭제 권한은 관리자만, 삭제는 7일 유예, 백업 30일, 로그 24개월, 가명처리(관리자만 복구 가능)
  - 재진단/팀 이동 시: 이전 자료 복구 후 활성 접근 기간 12개월 자동 연장
  - 복구 SLA(운영 기준): 요청 후 24~48시간 내 복구
  - 관리자 정의: FT 중 권한이 부여된 관리 FT

### 10.2 품질/가드레일
- 근거 없는 단정 금지, "확인 필요" 표기
- 톤 규정: 중립적·비판 없는 언어
- 외부 웹 검색 정보로 고객 팀 해석 금지

### 10.3 운영 제약
- 비개발자(FT)가 직접 업로드/생성/다운로드 가능해야 함
- 한국어 기본, 필요 시 다국어 병기 옵션

## 11. AI 출력 가이드
- 1순위: 업로드된 Belbin 보고서 원문
- 2순위: Team Context Data 및 인터뷰 노트
- 3순위: 컨설팅사 표준 템플릿/가이드
- 주요 주장/수치는 반드시 근거 출처 표시

## 12. 통합/연동 (MVP 기준)
- MVP: 이메일 + 인앱 알림
- 채널 연동: vNext (Teams/Slack 중 선택 TBD)
- 향후: 캘린더 연동

## 13. 오픈 이슈 / 결정 필요
- Belbin Interplace 접근 가능 여부
- Team Context Data 수집 항목/형태 확정
- 리포트 생성 품질 기준(정확도/신뢰도)
- 채널 연동 대상(Teams/Slack) 선택 시점 확정

## 14. 리스크
- 입력 데이터 품질 편차로 리포트 일관성 저하
- 과도한 자동화로 FT의 전문성/재량 감소 우려

## 15. 향후 확장 (vNext)
- 워크숍 설계 패키지(아젠다/활동/질문 카드/스크립트)
- 실행 넛지(ActionCard/체크인/리마인드)
  - ActionCard 필드(초안): 목적/실행 행동/빈도/DoD/파트너/기한/알림주기/오너 (MVP-3에서 확정)
- HRBP/조직장용 주간 브리프/임팩트 스냅샷 자동 생성
- 조직 단위 대시보드 및 멀티 팀 비교
- 워크숍 결과 보고서 자동화(PPT)
- 팀별 실행율/응답율 게임화 고도화
