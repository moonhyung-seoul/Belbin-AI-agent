# 벨빈 AI Agent MVP 기능 리스트 + 2인 로드맵 (MVP-1: 분석 리포트)

## 기준
- MVP-1 범위: 분석 리포트 생성(팀/개인/리더/FT 4종 패키지)
- 연동: 이메일 + 인앱 알림(Teams/Slack는 vNext)
- 개발 인력: 2명
- 스프린트: 2주 기준(총 6주 가정)

## 1) MVP-1 기능 리스트 (Must-Have)
### A. 프로젝트/권한/데이터 관리
- 프로젝트 생성/관리(팀 단위)
- 사용자 역할: FT/리더/구성원
- 접근 권한: 개인 상세는 FT 전용, 리더/구성원은 팀 리포트 접근 가능
- 파일 업로드: Belbin Team/Individual PDF, Team Context Data(XLSX), 인터뷰 노트
- 입력 폼: 팀 미션/업무/핵심 이슈/제약/이해관계자
- 자료 버전 관리(업로드 이력)

### B. 분석 리포트 생성
- PDF 텍스트 추출 및 구조화(팀/개인 보고서)
- Team Context Data 매핑(팀 업무/미션/KPI/이슈 등)
- 팀 리포트: 강점/약점/리스크 + 액션 안건 Long List
- 개인 리포트(팀원용): Self vs Observer, Top/Bottom 역할, 기여 포인트, 행동 제안
- 리더 리포트: 리더 분석 + 사후 코칭용 이슈 연결
- FT-only 리포트: 퍼실리테이션 주의점 + 참석자별 1-pager
- 근거 표시: 어떤 입력 자료의 어떤 항목/페이지 기반인지 표시
- "확인 필요" 라벨(근거 부족 시)
- FT 편집/승인 플로우: 편집 가능한 초안 → FT 수정 → 승인 후 PDF 확정

### C. 출력/공유
- 리포트 PDF 다운로드
- 권한 기반 공유 링크(리더/구성원)

## 2) MVP-1 범위에서 제외(명시적 아웃)
- Belbin 진단 자체 생성/평가
- 워크숍 설계/프로그램 패키지 생성
- 실행 넛지(ActionCard/체크인/리마인드)
- Slack/Teams/캘린더 연동
- 조직 단위 멀티 팀 비교 대시보드

## 3) 2인 기준 일정/로드맵 (6주, 예상)
### Sprint 0 (1주) - 기초 설계/환경 구축
- 요구사항 확정(권한/보안/데이터 보관 정책)
- 데이터 모델/권한 모델 설계
- 기본 UI 구조(프로젝트/업로드/리포트)

### Sprint 1 (2주) - 데이터 인입 & 구조화
- 파일 업로드/검증(PDF/XLSX)
- PDF 텍스트 추출 및 섹션 파싱
- Team Context Data 매핑
- 버전 관리/이력

### Sprint 2 (2주) - 리포트 생성/편집/승인
- 팀/개인/리더/FT 리포트 템플릿
- 근거 표시/"확인 필요" 처리
- FT 편집 화면 + 승인 흐름
- PDF 출력 1차

### Sprint 3 (1주) - 통합/QA/파일럿
- 권한/보안 점검
- 샘플 데이터로 파일럿
- 버그 수정 및 문서화

## 4) 다음 단계 (vNext)
