# 작업 분배표 + 이슈 목록 (MVP-1)

## 전제
- MVP-1 범위: 분석 리포트(팀/개인/리더/FT 4종) + FT 편집/승인 + PDF/공유 링크
- 인력: 2명

---

## 1) 역할 분배(추천)
### 역할 A: 데이터/리포트 엔진
- PDF 파싱/섹션 추출
- Team Context 매핑
- 리포트 JSON 생성(스키마 준수)
- 근거 표기/"확인 필요" 처리
- PDF 렌더링(템플릿 기반)

### 역할 B: UI/권한/편집
- 프로젝트 생성/업로드 UI
- FT 편집/승인 플로우 UI
- 리더/구성원 리포트 뷰
- 공유 링크/권한 처리
- 열람 로그/감사 기록

---

## 2) 이슈 목록(초안)
### A. 프로젝트/업로드
1. 프로젝트 생성 화면
- 입력: 프로젝트명/팀명/FT 담당자
- 완료 조건: 프로젝트 생성 후 상세 화면 진입

2. 팀 컨텍스트 입력 폼
- 필드: 미션/업무/이슈/제약/이해관계자(+선택: KPI/고객/의사결정)
- 완료 조건: 저장/수정 가능

3. 파일 업로드(팀/개인 PDF + TeamContext)
- 완료 조건: 업로드 성공/실패 상태 표시, 재업로드 가능

### B. 파싱/데이터 처리
4. Belbin PDF 텍스트 추출
- 완료 조건: 팀/개인 보고서 텍스트 추출 및 섹션 분리

5. Team Context 매핑
- 완료 조건: 입력 폼/엑셀 데이터를 리포트 템플릿에 매핑

### C. 리포트 생성
6. 리포트 JSON 생성(팀/개인/리더/FT)
- 완료 조건: 스키마 준수 JSON 생성

7. 근거 표기/"확인 필요" 처리
- 완료 조건: 모든 주요 항목에 source 표시 또는 needConfirm

8. 리포트 초안 미리보기
- 완료 조건: 팀/개인/리더/FT 탭 전환 확인

### D. FT 편집/승인
9. FT 편집 화면(블록 단위)
- 완료 조건: 텍스트/표 수정, 변경 이력 저장

10. 승인 플로우
- 완료 조건: 승인 후 PDF 확정, 버전 생성

### E. 배포/권한
11. 권한 매트릭스 적용
- 완료 조건: 리더/구성원 접근 범위 제한 확인

12. 공유 링크 생성/열람
- 완료 조건: 링크별 권한 유지, 만료 옵션(선택)

13. 열람 로그 기록
- 완료 조건: 리더/구성원 열람 시 로그 저장

### F. 출력
14. PDF 출력(4종)
- 완료 조건: 팀/개인/리더/FT 리포트 다운로드 가능

### G. 운영/보안
15. 데이터 보관/아카이브 정책 적용
- 완료 조건: 12개월 후 아카이브 전환 플래그 지원(운영 처리)

---

## 3) 우선순위 제안
- P0: 1~8, 9, 10, 11, 14
- P1: 12, 13, 15

---

## 4) 작업 순서(스프린트 기준)
- Sprint 0: 1,2,3 (기본 화면/입력)
- Sprint 1: 4,5,6,7 (파싱/매핑/JSON)
- Sprint 2: 8,9,10,14 (미리보기/편집/승인/PDF)
- Sprint 3: 11,12,13,15 (권한/배포/로그/정책)

---

## 5) 오픈 이슈(질문)
- Belbin 보고서 템플릿 표준화 범위(여러 버전 대응 필요?)
- PDF 템플릿 디자인 확정(브랜딩 기준)
- 공유 링크 만료/재발급 정책
